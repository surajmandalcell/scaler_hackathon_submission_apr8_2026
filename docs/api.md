# HTTP API

FundLens exposes two API surfaces from the same FastAPI app at port 7860.

## OpenEnv lifecycle endpoints

These follow the OpenEnv specification and are auto-generated by `create_app()` from `openenv-core`.

| Method | Path     | Body                              | Returns                                    |
|--------|----------|-----------------------------------|--------------------------------------------|
| POST   | `/reset` | `{"task_id": "easy\|medium\|hard"}` | `{observation, reward, done}`              |
| POST   | `/step`  | `{"tool_name": "...", "arguments": {...}}` | `{observation, reward, done}`     |
| GET    | `/state` | -                                 | Current `FundLensState`                    |
| GET    | `/health`| -                                 | `{"status": "healthy"}`                    |

### Observation shape

```json
{
  "observation": {
    "task_id": "easy",
    "difficulty": "easy",
    "task_description": "RE Alpha Fund I -- Q4 2024 reporting...",
    "available_funds": ["alpha"],
    "reporting_period": "Q4_2024",
    "portfolio_nav_usd": 42.3,
    "tool_result": null,
    "grading_result": null,
    "message": "LEVEL 1 -- NAV BRIDGE ONLY..."
  },
  "reward": null,
  "done": false
}
```

When you call `/step` with a tool, `tool_result` contains the tool's return value. When you submit via `submit_report`, `grading_result` contains the reward and per-item breakdown.

## REST endpoints (for the React frontend)

These are not part of the OpenEnv spec. They wrap the underlying data store for the dashboard UI.

| Method | Path                     | Description                                  |
|--------|--------------------------|----------------------------------------------|
| POST   | `/api/load-scenario`     | Query: `?task_id=easy\|medium\|hard`. Loads seed data into the API store. |
| GET    | `/api/portfolio`         | Returns `{fund_id: {fund_name, beginning_nav, ending_nav, moic, irr, reporting_date}}`. |
| GET    | `/api/bridge/{fund_id}`  | Returns the 8-line NAV bridge for one fund.  |
| GET    | `/api/deals/{fund_id}`   | Returns `{fund_id, deals: [...]}` with property, sector, ownership, invested, received. |
| GET    | `/api/cashflows/{fund_id}` | Returns aggregated totals + sorted cashflow records. Optional query: `?deal_id=...`. |
| GET    | `/api/sectors`           | Returns `{sector: {total_invested, total_received, deal_count}}` across all loaded funds. |

### Example: load and inspect

```bash
# Load the easy scenario
curl -X POST "http://localhost:7860/api/load-scenario?task_id=easy"
# {"task_id":"easy","funds_loaded":["alpha"],"fund_count":1}

# Get portfolio summary
curl http://localhost:7860/api/portfolio
# {"alpha":{"fund_name":"RE Alpha Fund I","beginning_nav":38.5,"ending_nav":42.3,...}}

# Get the NAV bridge
curl http://localhost:7860/api/bridge/alpha
# {"beginning_nav":38.5,"contribution":0,"disposition":5.5,"income":2.73,
#  "cashflow_adjusted_nav":35.73,"income_reversal":-2.73,"write_up_down":9.3,"ending_nav":42.3}
```

### Example: agent flow via OpenEnv

```bash
# 1. Reset
curl -X POST http://localhost:7860/reset \
  -H "Content-Type: application/json" \
  -d '{"task_id":"easy"}'

# 2. Get NAV bridge via MCP tool
curl -X POST http://localhost:7860/step \
  -H "Content-Type: application/json" \
  -d '{"tool_name":"get_nav_bridge","arguments":{"fund_id":"alpha"}}'

# 3. Submit the report
curl -X POST http://localhost:7860/step \
  -H "Content-Type: application/json" \
  -d '{"tool_name":"submit_report","arguments":{"nav_bridge":{"beginning_nav":38.5,...}}}'
```

The third call returns a `grading_result` with the reward.

## Static frontend

`GET /` serves the React `index.html`. `GET /assets/*` serves the bundled JS and CSS. Any other unmatched path also serves `index.html` so client-side routing works.
